## 🚀 快速开始

### 第一步：环境准备（30分钟）

```bash
# 1. 进入学习目录
cd ~/Desktop/学习/06-MySQL

# 2. 阅读环境准备文档
# 打开 00-环境准备.md

# 3. 启动MySQL（Docker方式推荐）
docker run --name mysql-study \
  -e MYSQL_ROOT_PASSWORD=123456 \
  -p 3306:3306 \
  -v ~/mysql-data:/var/lib/mysql \
  -d mysql:8.0 \
  --character-set-server=utf8mb4 \
  --collation-server=utf8mb4_unicode_ci

# 4. 连接MySQL
docker exec -it mysql-study mysql -uroot -p123456

# 5. 创建学习数据库
CREATE DATABASE mysql_study DEFAULT CHARACTER SET utf8mb4;
USE mysql_study;
```

## 💡 使用技巧

### 1. 命令行使用

```bash
# 连接MySQL
mysql -uroot -p

# 或使用Docker
docker exec -it mysql-study mysql -uroot -p123456

# 执行SQL文件（如果你保存了练习SQL）
mysql -uroot -p mysql_study < practice.sql
```

### 2. 查看实时日志

```bash
# 查看MySQL容器日志
docker logs -f mysql-study

# 查看慢查询日志
docker exec mysql-study tail -f /var/log/mysql/mysql-slow.log
```

### 3. 快速重置环境

```bash
# 删除并重建数据库
docker exec -it mysql-study mysql -uroot -p123456 -e "DROP DATABASE IF EXISTS mysql_study; CREATE DATABASE mysql_study;"

# 或重启容器
docker restart mysql-study
```

## ⚠️ 注意事项

### 安全提示

1. **不要在生产环境练习**
2. **定期备份学习数据**（虽然可以重建）
3. **Docker容器重启会丢失未持久化的配置**

### 性能提示

1. **适当调整buffer pool大小**：
   ```sql
   -- 查看当前配置
   SHOW VARIABLES LIKE 'innodb_buffer_pool_size';

   -- 建议配置为可用内存的50-80%
   ```

2. **监控磁盘空间**：
   ```bash
   df -h
   du -sh ~/mysql-data
   ```

### 学习提示

1. **不要死记硬背**：理解原理比记住参数重要
2. **多动手实践**：每个SQL都要亲自执行
3. **对比观察**：修改参数观察行为变化
4. **记录问题**：建立自己的问题和解答文档
