# MySQL CRUD 实战教程 - 业务开发篇

> 面向业务开发的系统实战教程，从数据库设计到CRUD实现

---

## 📖 课程定位

**适合人群**：
- 有一定编程基础，需要用MySQL开发业务系统
- 了解基本SQL语法，想系统学习业务开发中的实战技巧
- 想通过实际案例理解数据库设计和查询优化

**学习目标**：
- 掌握系统的数据库表设计方法
- 掌握常见业务场景的数据库设计
- 熟练编写CRUD操作的SQL
- 理解查询优化的基本原则
- 建立良好的SQL编写习惯

### 第六章：最佳实践

**23-SQL编写规范.md**
- 命名规范
  - 数据库名：小写、下划线分隔
  - 表名：小写、下划线分隔、复数/单数统一
  - 字段名：小写、下划线分隔、见名知意
  - 索引名：idx_表名_字段名（普通索引）、uniq_表名_字段名（唯一索引）
- SQL风格
  - 关键字大写 vs 小写（统一）
  - 缩进和换行
  - 注释规范（表注释、字段注释、SQL注释）
- 查询性能建议
  - 避免SELECT *
  - 避免N+1查询
  - 合理使用LIMIT
  - 避免隐式类型转换
- 安全规范
  - SQL注入防护（参数化查询、预编译）
  - 权限最小化
  - 敏感数据加密
  - 避免明文存储密码
- 可维护性
  - 复杂SQL拆分
  - 视图使用
  - 存储过程使用（慎用）
  - 数据字典维护

**24-常见陷阱与解决方案.md**
- NULL值陷阱
  - NULL的比较（NULL != NULL）
  - NULL的聚合函数（COUNT忽略NULL）
  - NULL的排序
  - NOT IN与NULL陷阱
- 字符串陷阱
  - 字符串比较大小写（排序规则）
  - 字符串截断
  - 字符编码问题
- 时间日期陷阱
  - 时区问题（DATETIME vs TIMESTAMP）
  - 日期格式化
  - 闰秒、闰年
  - 时间范围查询（边界问题）
- 数值陷阱
  - 浮点数精度问题
  - 金额存储（使用DECIMAL）
  - 整数溢出
  - 除零错误
- 类型转换陷阱
  - 隐式类型转换导致索引失效
  - 字符串与数字比较
  - 日期字符串比较
- 索引陷阱
  - OR条件索引失效
  - IN条件数量过多
  - 函数导致索引失效
  - 联合索引顺序错误
- 性能陷阱
  - SELECT * 性能问题
  - 子查询性能问题
  - JOIN过多表
  - 深度分页性能问题
- 并发陷阱
  - 脏读、不可重复读、幻读
  - 死锁
  - 库存超卖
  - 更新丢失

**25-数据库设计评审清单.md**
- 命名规范检查
  - 表名、字段名是否符合规范
  - 索引名是否符合规范
  - 是否有注释
- 字段设计检查
  - 字段类型是否合适
  - 字段长度是否合理
  - 是否使用NOT NULL
  - 默认值是否合理
  - 是否有冗余字段
- 索引设计检查
  - 主键是否合理
  - 唯一索引是否完整
  - 查询索引是否缺失
  - 索引是否过多
  - 联合索引顺序是否合理
- 表关系检查
  - 表关系是否清晰
  - 外键是否需要
  - 多对多是否有中间表
- 性能检查
  - 是否有大字段
  - 是否需要拆表
  - 是否有合理的冗余
  - 统计字段是否需要
- 扩展性检查
  - 是否考虑未来扩展
  - 是否有版本号
  - 是否有扩展字段
- 安全性检查
  - 敏感字段是否加密
  - 是否有权限控制
  - 是否防止SQL注入
